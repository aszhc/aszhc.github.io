使用CircleCI+GitHub 进行CI/CD

持续集成/持续部署 （CI/CD）是一种开发实践，允许软件团队在多个平台上更轻松、更快速地构建、测试和部署应用程序。CircleCI是一个流行的自动化平台，允许您为项目构建和维护 CI/CD 工作流。

CI

在CircleCI中设置CI

因此，我们的GitHub repo设置了，现在我们希望通过设置Circleci来启用CI。

我们需要创建一个配置文件，以便Circleci将知道我们想要的操作。

创建名为“.circleci”的文件夹，并在它的内部创建名为“config.yml”的文件。

1
2

mkdir .circleci
touch .circleci/config.yml

打开“.circleci/config.yml” ，修改内容如下（注释不用写）：

version: 2.1

jobs:
  build:	# 创建
    working_directory: ~/circleci-python	# 工作目录
    docker:	# 使用Docker
      - image: "circleci/python:3.6.4"
    steps:
      - checkout # 检查内容
      - run: python3 main.py # 运行创建命令
  test: # 测试
    working_directory: ~/circleci-python
    docker:
      - image: "circleci/python:3.6.4"
    steps:
      - checkout
      - run: python3 main-test.py # 运行测试命令

workflows: # 工作流程
  build_and_test:	# 起的名字：创建并测试 
    jobs:
      - build 	
      - test:	
          requires: # 测试的前提是创建
            - build

这里有两种类型的任务，我们希望CircleCI执行以下操作：运行main.py以及运行我们的测试文件。

在官方文档 CircleCI 文档 中可以学习到更加复杂的配置

编写config.yml后，进入CircleCI’s 网站，使用你的GitHub账号进行登录，并授权访问刚刚创建的circleci-demo仓库。

现在将刚创建的配置文件push到github，

Circleci应检测代码更改并为我们的分支运行我们的build_and_test工作流。 我们应该能够看到Circleci网站上的工作。

测试CircleCI是否工作

现在我们测试CircleCI，首先创建并切换到一条新的git分支add-test：

1

git checkout -b add-test

将如下代码加入main-test.py中的TestAdd()函数中：

1

assert Add(5, 5) == 10

push这条git分支到github：

1
2
3

git add .
git commit -m "add test to make sure 5+5 is equa to 10"
git push --set-upstream origin add-test

我们可以在github仓库看到这个分支请求，可以在circleci中看到正在运行的作业。

当然，CI（和单元测试）的有用部分正在确保我们的代码库正常工作。现在我们创建一条错误的测试，看circleci是否能够报错，在main-test.py中的TestAdd函数中添加如下代码：

1

assert Add(5, 5) == 11

将项目push到github：

1
2
3

git add .
git commit -m "Add error test to test circleci"
git push

我们可以发现circleci中发现并报错。

点击报红的标志可以发现出错的地方。

在github中如果circleci出错会阻止用户Merge pull request

CD

连续部署在很多方面都是有益的。它有助于标准化应用程序的部署步骤，并保护它免受未记录的更改。它还有助于避免执行重复步骤，并让您更专注于开发。借助 CircleCI，您可以跨所有不同的部署流程进行单一视图，以便进行开发、测试和生产。

package main

import "github.com/gin-gonic/gin"

func main() {
	r := gin.Default()
	r.GET("/", func(c *gin.Context) {
		c.JSON(200, gin.H{
			"message": "Welcome to the Go Registration API",
		})
	})

	r.POST("/register", func(c *gin.Context) {
		fullname := c.PostForm("fullname")
		phone := c.PostForm("phone")
		email := c.PostForm("email")

		c.JSON(200, gin.H{
			"status":   "success",
			"message":  "User details successfully posted",
			"fullname": fullname,
			"phone":    phone,
			"email":    email,
		})
	})
	r.Run()
}

这里我创建了两个端点：

  • /:发送一个GET请求，返回一个json字符串欢迎信息。
  • /register:发送一个POST请求，POST请求包含用户数据，返回json响应。

开启应用服务：

1

go run main.go

为了方便，不需要运行go run main.go, 我们可以创建makefile如下：

1
2

dev:
	go run main.go

现在我们可以使用make dev命令开启应用

:triangular_flag_on_post: 可以使用postman验证功能

使用Heroku部署

这里选择Heroku部署我们的go应用，在 Heroku dashboard 中新建一个Heroku应用程序，起名为go-registration-api或者其他。

还需要一个Heroku API key，位于您的帐户设置页面上。 您可以从下拉菜单中访问帐户设置。

将项目连接到CircleCI

  1. 首先将项目push到GitHub仓库，然后在CircleCI中添加该项目。

  2. 添加环境变量

    这里有两个环境变量需要添加：

    • HEROKU_APP_NAME：在这个例子中，heroku name 为go-registration-api
    • HEROKU_API_KEY：在帐户选项中上查找您的Heroku帐户API密钥

使用CircleCI自动化部署

首先在Go应用的根目录中创建一个Procfile（无扩展名），添加以下命令：

web: bin/go-registration-api

Heroku在bin目录中运行我们的Go应用程序的构建版本

现在我们配置CircleCI的配置文件，创建.circleci文件夹，并在此文件夹中创建config.yml，内容如下：

1
2
3
4
5
6
7

version: 2.1
orbs:
  heroku: circleci/heroku@0.0.10
workflows:
  heroku_deploy:
    jobs:
      - heroku/deploy-via-git

提交对项目的更改，然后推送到远程存储库以运行部署脚本。

这里如果不出意外，已经部署好了。

我们可以查看https://go-registration-api.herokuapp.com或者使用Postman，检测功能是否完整。
